# 画面プロトタイプビルダー (Screen Prototype Builder)

JSON定義ファイルをもとに、社内業務向けWeb画面のプロトタイプをリアルタイムで描画・確認し、スタンドアロンHTMLとしてエクスポートできるツールです。要件定義・レビュー・ステークホルダーへのデモを主な用途としています。

## 主な特徴

- **リアルタイムプレビュー**: JSON定義を編集するとプレビューが即座に更新されます。
- **AIによる自動生成**: AIへの自然言語入力（日本語）から画面定義JSONを自動生成できます（※Anthropic APIキーが必要）。
- **画面定義とデータ定義の一体管理**: 画面レイアウトとデータ仕様（フィールド定義）を単一のJSONで結びつけて管理できます。
- **UIテンプレート対応**: 検索一覧画面・詳細編集画面の2種類の業務画面テンプレートに標準対応しています。
- **エクスポート機能**: 単独で動作するプロトタイプHTMLファイルや、定義JSONファイルを出力可能です。
- **外部依存なし・環境構築不要**: 外部ライブラリやサーバー構築は不要です。単一のHTMLファイルを開くだけでローカル環境ですぐに動作します。

## 実行環境

ローカルのモダンブラウザ（Chrome / Edge / Safari / Firefox 最新版）で動作します。

## 使い方

1. `src/screen_prototype_builder.html` をブラウザで直接開きます。
2. 左パネルから目的のタブを選び、定義を作成します。
   - **📄 JSON タブ**: JSONを直接記述・編集して画面の定義を行います。
   - **📊 データタブ**: データフィールドと画面項目の対応関係や仕様を表形式で一覧確認できます。
   - **✨ AI生成タブ**: 自然言語で要件（例: 「取引先マスタの検索・登録画面が欲しい。項目は...」）を入力し、「画面定義を生成」ボタンを押すとAIがJSONを作成します。
3. 右側のプレビューエリアでリアルタイムに変更を確認します。画面上部のタブから作成した各画面を切り替えられます。
4. トップバー右上のボタンから、作成した定義データのJSONエクスポートや、現在表示中画面のHTMLエクスポートを行います。

## JSON定義構造の概要

画面は以下のようなJSONフォーマットで定義します。

```json
{
  "id":         "（文字列）定義の識別子",
  "title":      "（文字列）全体画面タイトル",
  "dataFields": [ /* スキーマ・データモデルのフィールド定義 */ ],
  "sampleData": [ /* 検索一覧やフォームに表示するサンプルデータ */ ],
  "screens":    [ /* search(検索画面) または form(編集画面) のUIレイアウト定義 */ ]
}
```
